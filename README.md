# Healthcare Analytics Dashboard (Client Requirement Based)

## Executive Summary
This project delivers a client-driven healthcare analytics dashboard built using Tableau. The solution was developed based on documented client requirements to provide actionable insights into healthcare performance metrics.

---

## Client Problem Statement
The client required a centralized analytics dashboard to:

- Monitor patient admission trends
- Track treatment and department performance
- Analyze operational efficiency metrics
- Identify patterns impacting healthcare service delivery

The objective was to convert raw healthcare data into meaningful business intelligence for management-level decision-making.

---

## Project Objectives
- Translate client requirements into analytical metrics
- Design interactive Tableau dashboard
- Enable drill-down analysis by department, time, and patient category
- Provide executive-level KPI visibility

---

## Tools & Technologies
- Tableau
- Excel (Data Preprocessing)
- Data Modeling
- Calculated Fields
- Interactive Filters & Parameters

---

## Development Approach

### 1. Requirement Analysis
- Reviewed client Word document
- Identified key KPIs and reporting needs
- Mapped requirements to measurable metrics

### 2. Data Preparation
- Cleaned and structured dataset
- Validated missing values
- Standardized healthcare metrics

### 3. Dashboard Development
- Designed KPI summary section
- Built trend analysis visualizations
- Created department-level performance comparison
- Added interactive filters for user-driven exploration

---

## Key Insights Generated
- Identified peak patient admission periods
- Highlighted high-performing and underperforming departments
- Detected operational bottlenecks
- Provided visibility into treatment distribution patterns

---

## Business Impact
The dashboard supports:
- Faster strategic decision-making
- Improved healthcare performance monitoring
- Operational transparency
- Data-backed resource allocation planning
